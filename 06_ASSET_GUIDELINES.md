|------|------|

| **px** | 픽셀 (화면의 가장 작은 점 단위) |

## 1. 게임 화면 구조

### 1.1 전투 화면 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│                    [고정 배경 이미지]                     │
│                                                         │
│   ┌─────┐ ┌─────┐ ┌─────┐       ┌─────┐ ┌─────┐ ┌─────┐ │
│   │ 적1 │ │ 적2 │ │ 적3 │  VS   │ 펫1 │ │ 펫2 │ │ 펫3 │ │
│   └─────┘ └─────┘ └─────┘       └─────┘ └─────┘ └─────┘ │
│                                        ┌─────────┐      │
│                                        │ 캐릭터  │      │
│                                        └─────────┘      │
├─────────────────────────────────────────────────────────┤
│  [HP/MP 바]  [행동 버튼: 공격/방어/스킬/아이템/포획/도주]   │
└─────────────────────────────────────────────────────────┘
```

### 1.2 핵심 특징

| 항목 | 설명 |
|------|------|
| **화면 방향** | 가로형 (Landscape) |
| **시점** | 사이드뷰 (옆에서 바라봄) |
| **진영 배치** | 왼쪽 = 적군, 오른쪽 = 아군 |
| **캐릭터 방향** | 좌우 2방향만 (상하 없음) |
| **기본 스프라이트** | 왼쪽을 바라봄 (←) |
| **배경** | 스테이지 테마별 고정 이미지 |

### 1.3 방향 처리

```
적군 (→ 오른쪽 봄)     아군 (← 왼쪽 봄)
  미러링 처리           기본 스프라이트
```

> **💡 미러링**: 1방향만 제작하고 코드에서 좌우 반전 → 에셋 제작량 50% 감소

## 2. 기본 규격

### 2.1 게임 기준 해상도

| 플랫폼 | 해상도 | 비율 |
|--------|--------|------|
| **PC** | 1920 × 1080px | 16:9 |
| **모바일** | 1280 × 720px | 16:9 |
| **최소** | 960 × 540px | 16:9 |

### 2.2 기본 단위

- 기준 단위: **32px**
- 모든 에셋은 32의 배수로 제작 권장
- 스케일링: 1x, 2x만 허용

### 2.3 파일 용량 기준

| 에셋 종류 | 권장 | 최대 |
|-----------|------|------|
| 배경 이미지 | 200KB | 500KB |
| 캐릭터 스프라이트 시트 | 30KB | 80KB |
| 펫 스프라이트 시트 | 30KB | 80KB |
| UI 스프라이트 시트 | 80KB | 150KB |
| 이펙트 스프라이트 시트 | 50KB | 100KB |

## 3. 배경 에셋

### 3.1 배경 사이즈

| 용도 | 사이즈 |
|------|--------|
| 전투 배경 | 1920 × 1080px |
| 모바일 대응 | 자동 스케일링 (중앙 기준 크롭) |

### 3.2 배경 테마 목록

| 테마 | 스테이지 범위 | 설명 |
|------|---------------|------|
| **초원** | 1~10 | 풀밭, 하늘, 나무 |
| **숲** | 11~20 | 울창한 숲, 나무 그늘 |
| **동굴** | 21~30 | 어두운 동굴, 광물 |
| **사막** | 31~40 | 모래, 선인장, 해골 |
| **설원** | 41~50 | 눈밭, 얼음, 눈보라 |
| **화산** | 51~60 | 용암, 바위, 연기 |

> **💡 스테이지 1~10은 모두 "초원" 배경 사용. 출현 몬스터만 다름!**

### 3.3 파일 예시

```
bg_grassland.png    ← 초원
bg_forest.png       ← 숲
bg_cave.png         ← 동굴
bg_desert.png       ← 사막
bg_snow.png         ← 설원
bg_volcano.png      ← 화산
```

## 4. 캐릭터 에셋

### 4.1 기본 사이즈

- 단일 프레임: **64 × 64px**
- 방향: **왼쪽을 바라봄 (←)** 1방향만 제작

### 4.2 필수 애니메이션 (총 16장)

| 동작 | 태그 | 프레임 수 |
|------|------|-----------|
| 대기 | `idle` | 4 |
| 공격 | `attack` | 4 |
| 피격 | `hurt` | 2 |
| 사망 | `death` | 3 |
| 승리 | `victory` | 3 |

### 4.3 스프라이트 시트 레이아웃 (256 × 128px)

```
┌────────┬────────┬────────┬────────┐
│ idle_1 │ idle_2 │ idle_3 │ idle_4 │ Row 0: 대기
├────────┼────────┼────────┼────────┤
│ atk_1  │ atk_2  │ atk_3  │ atk_4  │ Row 1: 공격
├────────┼────────┼────────┼────────┤
│ hurt_1 │ hurt_2 │ death_1│ death_2│ Row 2: 피격/사망
├────────┼────────┼────────┼────────┤
│ death_3│ vic_1  │ vic_2  │ vic_3  │ Row 3: 사망/승리
└────────┴────────┴────────┴────────┘
```

### 4.4 파일 예시

```
char_base_male.png      ← 기본 남성
char_base_male.json     ← 프레임 정보
char_base_female.png    ← 기본 여성
char_costume_001.png    ← 코스튬
```

## 5. 펫/몬스터 에셋

### 5.1 크기별 분류

| 크기 | 사이즈 | 예시 |
|------|--------|------|
| **S (소형)** | 48 × 48px | 새, 토끼, 도마뱀 |
| **M (중형)** | 64 × 64px | 늑대, 공룡새끼, 멧돼지 |
| **L (대형)** | 96 × 96px | 성체 공룡, 매머드 |
| **XL (특대)** | 128 × 128px | 보스 전용 |

### 5.2 방향 규칙

| 역할 | 방향 |
|------|------|
| 아군 펫 | ← 왼쪽 바라봄 (기본) |
| 적 몬스터 | → 오른쪽 바라봄 (미러링) |

### 5.3 필수 애니메이션 (총 21장)

| 동작 | 태그 | 프레임 수 |
|------|------|-----------|
| 대기 | `idle` | 4 |
| 공격 | `attack` | 4 |
| 기술1 | `skill_1` | 4 |
| 기술2 | `skill_2` | 4 |
| 피격 | `hurt` | 2 |
| 기쁨 | `happy` | 3 |

### 5.4 탑승 스프라이트 (별도 제작)

```
┌────────┬────────┬────────┬────────┐
│ m_idle1│ m_idle2│ m_idle3│ m_idle4│ 탑승 대기
├────────┼────────┼────────┼────────┤
│ m_atk1 │ m_atk2 │ m_atk3 │ m_atk4 │ 탑승 공격
└────────┴────────┴────────┴────────┘
```

### 5.5 파일 예시

```
pet_001_raptor_m.png          ← 001번 랩터 (중형)
pet_001_raptor_m.json
pet_001_raptor_m_mount.png    ← 탑승 버전
pet_001_raptor_m_rare.png     ← 희귀 컬러
pet_002_wolf_m.png
pet_003_trex_l.png            ← 대형
```

### 5.6 속성별 컬러 가이드

| 속성 | 주요 색상 | Hex |
|------|-----------|-----|
| **지(地)** | 초록 | #228B22 |
| **수(水)** | 파랑 | #1E90FF |
| **화(火)** | 빨강 | #FF4500 |
| **풍(風)** | 노랑 | #FFD700 |

## 10. 파일 네이밍 규칙

### 10.1 일반 규칙

- ✅ 모든 파일명 소문자
- ✅ 단어 구분은 언더스코어(_)
- ✅ 숫자는 3자리 패딩 (001, 002...)
- ✅ 확장자는 .png 또는 .json
- ❌ 한글, 공백, 특수문자 금지

### 10.2 접두사

| 접두사 | 카테고리 | 예시 |
|--------|----------|------|
| `bg_` | 배경 | bg_grassland.png |
| `char_` | 캐릭터 | char_base_male.png |
| `pet_` | 펫/몬스터 | pet_001_raptor_m.png |
| `wpn_` | 무기 | wpn_sword_001.png |
| `arm_` | 방어구 | arm_body_001.png |
| `icon_` | 아이콘 | icon_hp_24.png |
| `btn_` | 버튼 | btn_primary_md.png |
| `win_` | 윈도우 | win_default_9slice.png |
| `bar_` | 게이지/바 | bar_hp.png |
| `fx_` | 이펙트 | fx_hit_physical.png |

## 11. 폴더 구조

```
/assets/
├── /backgrounds/       ← 스테이지 배경
├── /characters/        ← 캐릭터
│   └── /costumes/
├── /pets/              ← 펫/몬스터
├── /equipment/         ← 장비 아이콘
│   ├── /weapons/
│   └── /armors/
├── /ui/                ← UI 요소
│   ├── /buttons/
│   ├── /windows/
│   ├── /bars/
│   └── /status/
├── /icons/             ← 아이콘
│   ├── /battle/
│   ├── /element/
│   └── /resource/
├── /effects/           ← 이펙트
└── /audio/             ← 사운드 (추후)
    ├── /bgm/
    └── /sfx/
```

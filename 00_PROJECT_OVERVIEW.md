## 4. 수익화 모델

### 과금 아이템 종류

| 카테고리 | 아이템 예시 | 게임 밸런스 영향 |
|----------|-------------|------------------|
| **포획 부스터** | 황금 포획석 (포획률 +20%) | 낮음 (확률 상승만) |
| **캐릭터 꾸미기** | 의상, 헤어, 이펙트 | 없음 (외형만) |
| **자동 사냥** | 자동사냥권 (1시간/1일) | 중간 (편의성) |
| **경험치 부스터** | EXP +50% (1시간) | 중간 (성장 가속) |
| **펫 슬롯 확장** | 기본 10마리 → 최대 20마리 | 낮음 |

### 과금 정책 원칙

1. **Pay to Win 최소화**: 과금으로 "더 강해지는 것"이 아닌 "더 빨라지는 것" 위주
2. **무과금 플레이 가능**: 모든 컨텐츠는 무과금으로 접근 가능
3. **확률형 아이템 최소화**: 뽑기보다는 직접 구매 위주

## 5. 기술 스택

### Frontend (클라이언트)

| 기술 | 버전 | 용도 |
|------|------|------|
| **React** | 18.x | UI 컴포넌트 관리 |
| **Phaser** | 3.x | 게임 렌더링 (전투, 애니메이션) |
| **Zustand** | 4.x | 전역 상태 관리 |
| **Vite** | 5.x | 빌드 도구 |
| **Tailwind CSS** | 3.x | 스타일링 |
| **TypeScript** | - | 타입 안정성 |

### Backend (서버)

| 기술 | 용도 |
|------|------|
| **Node.js + Express** | REST API 서버 |
| **Socket.io** | 실시간 통신 (전투 동기화, 파티) |
| **Redis** | 세션 관리, 실시간 데이터 캐싱 |

### Database

| 기술 | 용도 |
|------|------|
| **Supabase (PostgreSQL)** | 유저, 펫, 아이템 등 영구 데이터 |
| **Redis** | 현재 접속자 위치, 전투 상태 등 휘발성 데이터 |

### Deployment (배포)

| 서비스 | 용도 | 예상 비용 |
|--------|------|-----------|
| **Vercel** | 프론트엔드 호스팅 | 무료 (취미용) |
| **Railway** | 백엔드 서버 | $5~20/월 |
| **Supabase** | 데이터베이스 | 무료 (500MB까지) |
| **Cloudflare R2** | 이미지/에셋 저장 | $0.015/GB |

## 6. 개발 일정 (Phase 로드맵)

| Phase | 기간 | 목표 | 주요 기능 |
|-------|------|------|-----------|
| **Phase 1** | 2주 | 코어 전투 | 스탯, 레벨업, 기본 전투, 다굴, 크리티컬, 펫 운용/포획 |
| **Phase 2** | 2주 | 캐릭터/스테이지 | 캐릭터 생성, 속성 시스템, 스테이지 1~10, 별점 |
| **Phase 3** | 2주 | 전투 심화 | 상태이상 9종, 주술, 펫 스킬 |
| **Phase 4** | 2주 | 제작/NPC | 내구도, 수리, 제작, 상점, 펫 관리소 |
| **Phase 5** | 2주 | 멀티/폴리싱 | 요일 던전, 멀티플레이, 밸런싱, 버그 수정 |

**예상 총 개발 기간: 10주 (약 2.5개월)**
